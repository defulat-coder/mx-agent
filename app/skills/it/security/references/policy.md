# 马喜公司信息安全管理制度

## 一、密码策略

| 项目 | 要求 |
|------|------|
| 最小长度 | 8 位 |
| 复杂度 | 须含大写字母、小写字母、数字，建议包含特殊字符 |
| 有效期 | 90 天强制更换 |
| 历史记录 | 不可与前 5 次密码相同 |
| 锁定策略 | 连续 5 次错误密码，锁定 30 分钟 |

### 密码使用规范
- 禁止将密码以明文形式记录（便签纸、文本文件等）
- 禁止多人共用同一账号
- 禁止将公司账号密码用于外部网站
- 发现密码泄露须立即修改并上报

## 二、数据分级保护

| 级别 | 说明 | 示例 | 要求 |
|------|------|------|------|
| 机密 | 泄露将造成重大损失 | 薪资数据、客户合同、核心代码 | 加密存储、禁止外发 |
| 内部 | 仅限公司内部使用 | 内部报告、流程文档 | 不可发送至外部邮箱 |
| 公开 | 可对外公开 | 官网内容、招聘信息 | 无特殊限制 |

### 数据保护要求
- 机密数据禁止通过即时通讯工具传输
- 离开工位须锁定电脑（Win: `Win+L`，Mac: `Ctrl+Command+Q`）
- 敏感文件需加密后再通过邮件发送
- 禁止将公司数据上传至个人网盘

## 三、外设与存储介质

- **U 盘**：仅允许使用公司配发的加密 U 盘
- **个人设备**：禁止连接公司内网（BYOD 需申请并安装 MDM）
- **外部硬盘**：需 IT 管理员审批后方可使用
- **打印/扫描**：含机密信息的文件需及时取走，禁止遗留在公共打印区

## 四、安全事件上报

### 需要上报的事件
- 收到可疑邮件（钓鱼邮件、恶意附件）
- 电脑感染病毒或勒索软件
- 发现账号被他人使用
- 机密数据意外泄露
- 设备丢失（笔记本、手机、U 盘等）

### 上报流程
1. **立即**联系 IT 管理员或创建紧急工单
2. 不要自行处理（如不要删除可疑文件，保留现场）
3. IT 管理员在 30 分钟内响应
4. 必要时启动应急预案

### 违规处理
- 首次违规：警告 + 安全培训
- 再次违规：通报批评
- 造成损失：按公司规定追究责任

## 五、软件安全

- 仅允许安装公司白名单内的软件
- 需要安装新软件须创建「软件安装」工单
- 禁止安装破解/盗版软件
- 禁止关闭杀毒软件和防火墙
